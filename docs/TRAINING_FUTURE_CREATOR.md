# 优创未来轻量版学生培训文档

## 1. 课程目标

本培训文档指导学生使用"优创未来轻量版"在浏览器里完成全部 AI 训练，并导出比赛材料包。

完成本课程后，学生应能做到：

- 在 Windows 学生机上安装并启动优创未来轻量版。
- 用四步工作流（准备数据 → 训练模型 → 测试效果 → 导出材料）完成每个任务。
- 在浏览器中上传图片或粘贴传感器数据完成 AI 训练与测试。
- 导出含训练好模型的比赛材料包，并迁移到行空板 M10。
- 使用 DFRobot 开源硬件外设完成摄像头、语音、传感器或执行器交互。
- 整理比赛需要的创作说明、演示视频和实物照片。

统一硬件基线：

- 主控：行空板 M10
- 图像输入：行空板 M10 摄像头或 DFRobot 摄像头模块
- 语音互动：行空板 M10 麦克风输入，DFRobot 语音合成或兼容播报外设输出
- 传感器与执行器：DFRobot 开源硬件外设

## 2. 课前准备

### 2.1 软件准备

学生电脑需要：

- Windows 10 或 Windows 11
- Python 3.11 或 3.12
- 浏览器：Edge 或 Chrome
- 优创未来轻量版安装包：`FutureCreator-Windows-Setup.zip`

### 2.2 Python 检查

按 `Win + R`，输入 `cmd`，打开命令行，输入：

```bat
python --version
```

如果能看到 `Python 3.11.x` 或 `Python 3.12.x`，说明 Python 可用。

### 2.3 硬件准备

每组建议准备：

- 行空板 M10 一块
- USB 数据线
- 摄像头模块
- DFRobot 语音合成模块或扬声器外设
- DFRobot 传感器模块，例如温度、距离、按钮、光线或心率类模块
- DFRobot 执行器模块，例如舵机、马达、灯光或蜂鸣器
- 比赛卡片或自制训练卡片

## 3. 安装优创未来轻量版

### 3.1 解压安装包

1. 找到 `FutureCreator-Windows-Setup.zip`。
2. 右键选择"全部解压"。
3. 进入解压后的 `lite-deeplearning-studio` 文件夹。

必须先解压，再运行安装脚本。

### 3.2 一键安装

双击：

```text
一键安装.bat
```

安装脚本会自动：

- 创建本地运行环境 `.venv`
- 安装网页应用依赖
- 安装图像识别、文本处理和传感器决策所需的轻量 AI 依赖
- 创建桌面快捷方式

macOS 用户双击：

```text
一键安装.command
```

### 3.3 启动软件

双击：

```text
启动软件.bat
```

macOS 用户双击：

```text
启动软件.command
```

或双击桌面快捷方式：

```text
优创未来轻量版
```

浏览器打开：

```text
http://127.0.0.1:8000
```

正常情况下，首页只显示"优创未来"任务，不显示"智能博物"。

## 4. 软件使用总流程

每个任务都在浏览器中按四步完成：

```
准备数据  →  训练模型  →  测试效果  →  导出材料
```

**完整操作步骤：**

1. 打开首页（工作台），在「竞赛专区」点击「优创未来」，再选择任务卡片。
2. 填写项目名称和学生姓名，确认目标硬件为"行空板 M10 + DFRobot 开源硬件外设"。
3. 点击"创建项目，开始训练"，进入四步工作流页面。
4. **第 1 步「准备数据」**：在浏览器中添加类别并上传图片，或粘贴传感器 CSV 表格，点击"保存数据"。
5. **第 2 步「训练模型」**：点击"开始训练"，模型在电脑本地训练，数秒内完成，页面显示准确率。
6. **第 3 步「测试效果」**：上传一张新图片或输入一行传感器数值，实时查看模型预测结果。
7. **第 4 步「导出材料」**：点击"导出比赛材料包"，下载 zip 文件，包含训练好的模型、全部数据、可运行脚本、硬件说明和比赛清单。
8. 解压导出包，运行 `python run.py` 验证。
9. 查看 `outputs/ai_validation_result.json`，确认 `"status": "ok"`。
10. 按 `hardware/README.md` 迁移到行空板 M10，连接 DFRobot 外设展示。
11. 按 `submission/README.md` 整理提交材料。

> **关键点**：AI 模型在浏览器里训练好后会随导出包一起打包，不需要在命令行单独运行训练脚本。

## 5. 小学组：语音互动与单类图像识别

### 5.1 比赛目标

小学组通常要求：

- 完成 1 条基于关键词识别的人机语音互动。
- 在 4 类卡片中自选 1 类完成识别。
- 可以结合语音合成、语音交互控制或简单运动控制。

### 5.2 选择任务

在「优创未来」竞赛页点击任务卡：

```text
小学组：语音互动与单类图像识别
```

### 5.3 第 1 步：准备数据

在「准备数据」面板：

- 点击"添加类别"，输入 1 类现场卡片的名称（如"医疗卡片"）。
- 为该类别上传 5 到 8 张清晰卡片图片。
- 同时记录关键词（如"开始导诊"、"停止导诊"），后续在 `speech/voice_config.json` 中配置。
- 点击"保存数据"。

### 5.4 第 2 步：训练模型

点击"开始训练"，等待准确率显示。单类任务准确率通常接近 100%。

### 5.5 第 3 步：测试效果

上传一张未在训练集中的同类卡片，确认模型能正确识别。

### 5.6 第 4 步：导出材料

- 点击"导出比赛材料包"，下载 zip。
- 解压后重点检查：
  - `models/image_classifier.joblib`（已训练的模型）
  - `speech/voice_config.json`（关键词配置，按现场修改）
  - `speech/speech_output.py`（行空板关键词识别模板）
  - `hardware/README.md`（行空板 M10 + DFRobot 外设迁移说明）

### 5.7 电脑端验证

在任务包目录运行：

```bat
python run.py
```

成功后检查 `outputs/ai_validation_result.json` 中 `"status": "ok"`。

### 5.8 迁移到行空板 M10

迁移重点：

- 行空板 M10 使用麦克风识别关键词，命中后启动图像识别流程。
- 摄像头拍摄卡片，模型输出识别结果。
- DFRobot 语音或灯光外设给出反馈。

演示时说清楚：关键词是什么，识别的是哪一类卡片，行空板 M10 和 DFRobot 外设分别负责什么。

## 6. 初中组：视觉模型训练与调用

### 6.1 比赛目标

初中组通常要求：

- 完成 1 条关键词语音互动。
- 在 4 类卡片中自选 2 类完成识别。
- 体现视觉模型训练和模型调用。

### 6.2 选择任务

在「优创未来」竞赛页点击任务卡：

```text
初中组：视觉模型训练与调用
```

### 6.3 第 1 步：准备数据

在「准备数据」面板：

- 点击"添加类别"，分别添加 2 类卡片（如"医疗卡片"、"器械卡片"）。
- 每类上传 5 到 8 张图片（建议不同角度/光线各 2 到 3 张）。
- 点击"保存数据"。

> 每类至少 2 张才能开始训练，≥5 张准确率更稳定。

### 6.4 第 2 步：训练模型

点击"开始训练"，等待准确率显示（目标 ≥ 85%）。

- 准确率偏低时，检查两类图片是否有外观相似的情况，增加图片数量后重新训练。

### 6.5 第 3 步：测试效果

分别上传两类卡片各 1 张新图，确认模型能正确区分。

### 6.6 第 4 步：导出材料

- 点击"导出比赛材料包"，下载 zip。
- 解压后检查：
  - `models/image_classifier.joblib`
  - `ai_runtime/core.py`（行空板上调用模型的核心接口）
  - `docs/ai_validation.md`（训练验证说明）

### 6.7 电脑端验证

```bat
python run.py
```

检查 `outputs/ai_validation_result.json` 确认 2 类都能预测。

### 6.8 迁移到行空板 M10

- 将导出包复制到行空板 M10。
- 摄像头采集图片，关键词触发识别。
- 屏幕显示识别结果，DFRobot 外设播报或执行反馈动作。

## 7. 高中组：大模型语音互动、视觉识别与运动控制

### 7.1 比赛目标

高中组通常要求：

- 基于人工智能大模型的人机语音互动。
- 智能体角色与现场任务主题相符。
- 在 4 类卡片中自选 2 类完成识别。
- 使用机械装置自主夹取 3 个轻质立方体并摆放。

轻量版当前提供：

- 图像识别训练与调用模板。
- 行空板 M10 语音接口模板。
- DFRobot 外设和运动控制接口说明。
- 大模型服务扩展位（学生后续接入真实大模型 API）。

### 7.2 选择任务

在「优创未来」竞赛页点击任务卡：

```text
高中组：大模型语音互动、视觉识别与运动控制
```

### 7.3 第 1 步：准备数据

与初中组相同：选 2 类卡片，每类上传 5 张以上图片，保存数据。

### 7.4 第 2 步：训练模型

点击"开始训练"，等待准确率 ≥ 85%。

### 7.5 第 3 步：测试效果

上传两类卡片各 1 张新图，确认识别正确。

### 7.6 第 4 步：导出材料

- 点击"导出比赛材料包"，下载 zip。
- 解压后重点关注：
  - `models/image_classifier.joblib`
  - `speech/speech_output.py`（语音接口，后续替换为大模型 API）
  - `hardware/README.md`（运动控制接口位置说明）

### 7.7 角色设计与运动控制

高中组需要清楚的智能体角色，示例：

```text
我是智慧医疗助手，可以根据识别到的医疗卡片，给出任务提示，并控制机械装置完成取放动作。
```

建议视觉识别流程：语音关键词触发 → 摄像头拍摄 → 模型识别 → 程序选择动作 → DFRobot 舵机执行。

先做最小闭环（识别卡片 → 输出动作编号 → 舵机执行固定动作），再逐步增加夹取和摆放。

## 8. 图像识别快速模板

### 8.1 适用场景

当比赛现场发布卡片类别后，可以快速生成一个图像识别项目。适合各组别在训练营或比赛现场临时使用。

### 8.2 选择任务

在「优创未来」竞赛页点击任务卡：

```text
图像识别快速模板
```

### 8.3 第 1 步：准备数据

按现场公布的卡片类别添加类别并上传图片：

- 小学组：选 1 类，每类 ≥ 3 张。
- 初中/高中：选 2 类，每类 ≥ 5 张。

### 8.4 第 2 步到第 4 步

与各组别流程相同：训练 → 测试 → 导出。

### 8.5 常见问题

如果识别不准：

- 增加图片数量。
- 保持背景简洁（纯色背景最佳）。
- 调整拍摄光线，避免过暗或过曝。
- 避免两类卡片外观过于接近。

## 9. 传感器决策程序模板

### 9.1 比赛目标

围绕具身智能或智慧医疗，读取传感器输入，判断状态并输出动作。

示例：

```text
温度偏高 -> 提醒就诊
距离过近 -> 停止移动
按钮按下 -> 启动导诊
```

### 9.2 选择任务

在「优创未来」竞赛页点击任务卡：

```text
传感器决策程序模板
```

### 9.3 第 1 步：准备数据

在「准备数据」面板，粘贴传感器 CSV 数据（第一行为表头，最后一列为动作标签）：

```csv
temperature,distance,signal,action
38.6,12,1,提醒就诊
36.7,45,0,继续观察
37.8,20,1,建议复测
36.5,60,0,继续观察
39.1,8,1,提醒就诊
```

- 列名可以自定义，最后一列必须是动作标签。
- 每种动作至少准备 2 行数据。

### 9.4 第 2 步：训练模型

点击"开始训练"，系统会训练决策树模型，并显示决策规则（如"温度 > 38 → 提醒就诊"）。

### 9.5 第 3 步：测试效果

在测试框输入一行新的传感器数值，确认模型能输出正确的动作判断。

### 9.6 第 4 步：导出材料

- 点击"导出比赛材料包"，下载 zip。
- 解压后检查 `models/sensor_decision_model.joblib`。
- 运行 `python run.py`，检查 `outputs/ai_validation_result.json`。

### 9.7 迁移到行空板 M10

迁移重点：

- 用 DFRobot 传感器读取真实输入，整理成模型需要的字段格式。
- 调用决策模型输出动作编号。
- 用 DFRobot 执行器完成灯光、舵机、声音或提示动作。

## 11. 课堂训练建议

建议分三次课完成：

### 第一次课：软件和样例跑通

- 安装优创未来轻量版。
- 选择图像识别快速模板，上传 2 类各 3 张样例图片。
- 在浏览器内完成训练和测试，再导出并运行 `python run.py`。
- 理解 `models/` 和 `outputs/` 目录的内容。

### 第二次课：真实数据替换

- 采集现场卡片图片，上传替换样例图片。
- 重新训练，比较准确率变化。
- 重新导出材料包，验证模型可用。

### 第三次课：硬件展示

- 接入行空板 M10 摄像头或传感器。
- 接入 DFRobot 外设，完成一次现场演示。
- 录制演示视频，整理提交材料。

## 12. 现场演示检查表

演示前逐项检查：

- 行空板 M10 能正常开机。
- 摄像头画面清晰。
- DFRobot 传感器能读到数值。
- DFRobot 执行器能响应动作。
- 语音关键词已经替换成现场关键词。
- 图片类别已经替换成现场卡片类别并重新训练。
- `python run.py` 已经通过。
- 任务包中的核对清单已完成。
- 创作说明、实物照片和演示视频已备份。

## 13. 常见问题

### 13.1 只看到智能博物任务

说明下载错安装包了。优创未来应下载：

```text
FutureCreator-Windows-Setup.zip
```

### 13.2 语音关键词没有触发

检查：

- 关键词是否和现场公布的一致。
- 环境是否太吵。
- 行空板 M10 麦克风是否可用。
- 先用文本输入模式调试，确认逻辑正确后再接入语音。

### 13.3 图像识别不准

解决方法：

- 每类增加图片，目标 ≥ 5 张。
- 图片要清晰，卡片尽量占据画面中央。
- 避免背景干扰，训练数据要接近现场光线。
- 重新进入「准备数据」添加更多图片后，再次点击"开始训练"。

### 13.4 传感器决策不符合预期

检查：

- CSV 字段名是否和训练时一致。
- 数值单位是否统一（如温度单位要统一用摄氏度）。
- `action` 列是否写成了固定类别（不能用连续数值）。
- DFRobot 传感器接线是否稳定。

### 13.5 导出包里没有模型文件

必须先完成「训练模型」步骤，点击"开始训练"看到准确率后再导出。

## 14. 最终目标

完成训练后，学生应该能展示：

- 行空板 M10 接收语音或传感器输入。
- AI 模型完成图像识别或决策判断。
- DFRobot 外设完成播报、灯光、舵机或其他动作。
- 屏幕显示当前任务状态。
- 提交材料完整，能说明作品设计和 AI 技术点。
