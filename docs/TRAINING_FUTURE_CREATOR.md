# 优创未来轻量版学生培训文档

## 1. 课程目标

本培训文档用于指导学生使用“优创未来轻量版”完成优创未来比赛任务。

完成本课程后，学生应能做到：

- 在 Windows 学生机上安装并启动优创未来轻量版。
- 使用平台生成图像识别、语音互动、传感器决策和高中组方案任务包。
- 用样例数据或真实采集数据完成本地 AI 训练与推理。
- 将程序迁移到行空板 M10。
- 使用 DFRobot 开源硬件外设完成摄像头、语音、传感器或执行器交互。
- 整理比赛需要的创作说明、演示视频和实物照片。

统一硬件基线：

- 主控：行空板 M10
- 图像输入：行空板 M10 摄像头或 DFRobot 摄像头模块
- 语音互动：行空板 M10 麦克风输入，DFRobot 语音合成或兼容播报外设输出
- 传感器与执行器：DFRobot 开源硬件外设

## 2. 课前准备

### 2.1 软件准备

学生电脑需要：

- Windows 10 或 Windows 11
- Python 3.11 或 3.12
- 浏览器：Edge 或 Chrome
- 优创未来轻量版安装包：`FutureCreator-Windows-Setup.zip`

### 2.2 硬件准备

每组建议准备：

- 行空板 M10 一块
- USB 数据线
- 摄像头模块
- DFRobot 语音合成模块或扬声器外设
- DFRobot 传感器模块，例如温度、距离、按钮、光线或心率类模块
- DFRobot 执行器模块，例如舵机、马达、灯光或蜂鸣器
- 比赛卡片或自制训练卡片

## 3. 安装优创未来轻量版

### 3.1 解压安装包

1. 找到 `FutureCreator-Windows-Setup.zip`。
2. 右键选择“全部解压”。
3. 进入解压后的 `lite-deeplearning-studio` 文件夹。

必须先解压，再运行安装脚本。

### 3.2 一键安装

双击：

```text
一键安装.bat
```

安装脚本会自动：

- 创建本地运行环境 `.venv`
- 安装网页应用依赖
- 安装图像识别、文本处理和传感器决策所需的轻量 AI 依赖
- 创建桌面快捷方式

### 3.3 启动软件

双击：

```text
启动软件.bat
```

或双击桌面快捷方式：

```text
优创未来轻量版
```

浏览器打开：

```text
http://127.0.0.1:8000
```

正常情况下，首页只显示“优创未来”任务，不显示“智能博物”。

## 4. 软件使用总流程

每个任务都按同样流程完成：

1. 打开优创未来轻量版。
2. 选择任务。
3. 填写项目名称和学生姓名。
4. 确认目标硬件为“行空板 M10 + DFRobot 开源硬件外设”。
5. 点击“创建项目，开始训练”进入项目页。
6. 第 1 步「准备数据」：上传各类别图片或粘贴传感器表格。
7. 第 2 步「训练模型」：点击训练，查看准确率。
8. 第 3 步「测试效果」：上传新图片或输入数值实时测试。
9. 第 4 步「导出材料」：下载 zip 比赛材料包并解压。
10. 运行 `python run.py`。
11. 查看 `outputs/ai_validation_result.json`。
12. 按 `hardware/README.md` 迁移到行空板 M10。
13. 按 `submission/README.md` 整理提交材料。

## 5. 小学组：语音互动与单类图像识别

### 5.1 比赛目标

小学组通常要求：

- 完成 1 条基于关键词识别的人机语音互动。
- 在 4 类卡片中自选 1 类完成识别。
- 可以结合语音合成、语音交互控制或简单运动控制。

### 5.2 在平台中选择任务

选择：

```text
小学组：语音互动与单类图像识别
```

### 5.3 表单填写建议

项目名称：

```text
小学组智慧导诊识别
```

类别或关键词：

```text
开始导诊, 停止导诊
```

数据说明：

```text
现场发布关键词后替换关键词；从 4 类卡片中选择 1 类进行识别训练。
```

### 5.4 生成后检查

任务包中重点看：

- `data_sample/images/`：图像样例
- `speech/voice_config.json`：关键词配置
- `speech/speech_output.py`：行空板关键词识别模板
- `hardware/README.md`：行空板 M10 + DFRobot 外设迁移说明

### 5.5 电脑端验证

在任务包目录运行：

```bat
python run.py
```

成功后检查：

- `models/image_classifier.joblib`
- `outputs/predictions.json`
- `outputs/ai_validation_result.json`

### 5.6 迁移到行空板 M10

迁移重点：

- 行空板 M10 使用麦克风识别关键词。
- 命中关键词后启动图像识别流程。
- 摄像头拍摄卡片。
- 模型输出识别结果。
- DFRobot 语音或灯光外设给出反馈。

现场演示时要说清楚：

- 关键词是什么。
- 识别的是哪一类卡片。
- 行空板 M10 和 DFRobot 外设分别负责什么。

## 6. 初中组：视觉模型训练与调用

### 6.1 比赛目标

初中组通常要求：

- 完成 1 条关键词语音互动。
- 在 4 类卡片中自选 2 类完成识别。
- 体现视觉模型训练和模型调用。

### 6.2 在平台中选择任务

选择：

```text
初中组：视觉模型训练与调用
```

### 6.3 表单填写建议

项目名称：

```text
初中组视觉模型调用
```

类别或关键词：

```text
开始识别, 医疗卡片, 器械卡片
```

数据说明：

```text
选择 2 类卡片，每类至少准备 5 张图片，训练后由语音关键词触发识别。
```

### 6.4 数据准备

建议每类卡片拍摄：

- 正面清晰图片 3 张
- 轻微旋转图片 2 张
- 不同光线图片 2 张

图片命名建议：

```text
类别A_01.png
类别A_02.png
类别B_01.png
类别B_02.png
```

### 6.5 电脑端验证

运行：

```bat
python run.py
```

检查：

- 训练是否成功。
- 预测结果是否能区分 2 类卡片。
- `docs/ai_validation.md` 中的验证说明是否完成。

### 6.6 迁移到行空板 M10

迁移重点：

- 将训练好的模型和 `ai_runtime/` 复制到行空板 M10。
- 摄像头采集图片。
- 关键词触发识别。
- 屏幕显示识别结果。
- DFRobot 外设播报或执行反馈动作。

## 7. 高中组：大模型语音互动、视觉识别与运动控制

### 7.1 比赛目标

高中组通常要求：

- 基于人工智能大模型的人机语音互动。
- 智能体角色与现场任务主题相符。
- 在 4 类卡片中自选 2 类完成识别。
- 使用机械装置自主夹取 3 个轻质立方体并摆放。

轻量版当前提供：

- 图像识别训练与调用模板。
- 行空板 M10 语音接口模板。
- DFRobot 外设和运动控制接口说明。
- 大模型服务扩展位。

真实大模型和机械闭环需要学生后续结合现场硬件完成。

### 7.2 在平台中选择任务

选择：

```text
高中组：大模型语音互动、视觉识别与运动控制
```

### 7.3 表单填写建议

项目名称：

```text
高中组智慧医疗助手
```

类别或关键词：

```text
开始任务, 药品卡片, 器械卡片
```

数据说明：

```text
行空板 M10 负责语音触发和视觉识别；DFRobot 舵机或机械结构完成夹取动作；大模型服务用于角色对话。
```

### 7.4 角色设计

高中组需要一个清楚的智能体角色。

示例：

```text
我是智慧医疗助手，可以根据识别到的医疗卡片，给出任务提示，并控制机械装置完成取放动作。
```

角色设计要符合现场主题，不能只是泛泛聊天。

### 7.5 视觉识别流程

建议流程：

1. 语音关键词触发任务。
2. 摄像头拍摄卡片。
3. 图像模型识别卡片类别。
4. 程序根据类别选择动作。
5. DFRobot 舵机或执行器完成夹取。
6. 行空板屏幕显示当前步骤。

### 7.6 运动控制接口

轻量任务包不会直接控制所有机械结构，但会给出接口位置。

学生需要根据实际结构补充：

- 舵机角度
- 抓取顺序
- 放置位置
- 失败重试方式
- 安全停止按钮

建议先做最小闭环：

```text
识别卡片 -> 输出动作编号 -> 舵机执行一个固定动作
```

再逐步增加夹取和摆放。

## 9. 传感器决策程序模板

### 9.1 比赛目标

围绕具身智能或智慧医疗，读取传感器输入，判断状态并输出动作。

示例：

```text
温度偏高 -> 提醒就诊
距离过近 -> 停止移动
按钮按下 -> 启动导诊
```

### 9.2 在平台中选择任务

选择：

```text
传感器决策程序模板
```

### 9.3 真实数据格式

可以粘贴传感器 CSV：

```csv
temperature,distance,signal,action
38.6,12,1,提醒就诊
36.7,45,0,继续观察
37.8,20,1,建议复测
```

字段含义：

- `temperature`：温度或健康指标
- `distance`：距离或空间状态
- `signal`：按钮、开关、报警信号等
- `action`：程序要输出的动作

### 9.4 电脑端验证

运行：

```bat
python run.py
```

检查：

- `models/sensor_decision_model.joblib`
- `outputs/predictions.json`
- `outputs/ai_validation_result.json`

### 9.5 迁移到行空板 M10

迁移重点：

- 用 DFRobot 传感器读取真实输入。
- 将输入值整理成模型需要的字段。
- 调用决策模型输出动作。
- 用 DFRobot 执行器完成灯光、舵机、声音或提示动作。

## 11. 课堂训练建议

建议分三次课完成：

### 第一次课：软件和样例跑通

- 安装优创未来轻量版。
- 生成图像识别快速模板。
- 运行 `python run.py`。
- 理解 `models/` 和 `outputs/`。

### 第二次课：真实数据替换

- 采集卡片图片并上传到各类别。
- 重新训练模型，比较准确率变化。
- 重新导出比赛材料包。
- 比较样例数据和真实数据结果。

### 第三次课：硬件展示

- 接入行空板 M10 摄像头或传感器。
- 接入 DFRobot 外设。
- 完成一次现场演示。
- 录制演示视频。

## 12. 现场演示检查表

演示前逐项检查：

- 行空板 M10 能正常开机。
- 摄像头画面清晰。
- DFRobot 传感器能读到数值。
- DFRobot 执行器能响应动作。
- 语音关键词已经替换成现场关键词。
- 图片类别已经替换成现场卡片类别。
- `python run.py` 已经通过。
- 任务包中的核对清单已完成。
- 创作说明、实物照片和演示视频已备份。

## 13. 常见问题

### 13.1 只看到智能博物任务

说明下载错安装包了。

优创未来应下载：

```text
FutureCreator-Windows-Setup.zip
```

### 13.2 语音关键词没有触发

检查：

- 关键词是否和现场公布的一致。
- 环境是否太吵。
- 行空板 M10 麦克风是否可用。
- 是否先用文本输入调试过 `recognize_keyword`。

### 13.3 图像识别不准

解决方法：

- 每类增加图片。
- 图片要清晰。
- 卡片尽量占据画面中央。
- 避免背景干扰。
- 训练数据要接近现场光线。

### 13.4 传感器决策不符合预期

检查：

- CSV 字段名是否正确。
- 数值单位是否统一。
- `action` 是否写成了固定类别。
- DFRobot 传感器接线是否稳定。

## 14. 最终目标

完成训练后，学生应该能展示：

- 行空板 M10 接收语音或传感器输入。
- AI 模型完成图像识别或决策判断。
- DFRobot 外设完成播报、灯光、舵机或其他动作。
- 屏幕显示当前任务状态。
- 提交材料完整，能说明作品设计和 AI 技术点。
