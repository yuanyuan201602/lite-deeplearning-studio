# 算法全集目录（按六类任务）

本文件列出平台六类任务下**所有值得纳入的主流算法**，作为"先列全集、再选子集"的全集来源，也是训练页**算法卡片**的内容母本。

> 实现进度（v0.6.2）：四个分类任务的训练页已按本目录渲染**全集卡片**。经典且 CPU 可训练的算法（逻辑回归、朴素贝叶斯、随机森林、K近邻、决策树、**梯度提升 GBDT**、**多层感知机 MLP**、**支持向量机 SVM**[传感器]）可真正训练；深度学习卡片以"需显卡/展示"置灰，等显卡检测接入后自动解锁。

## 标注图例

每个算法标注三组信息：

- **训练算力**：`CPU` 纯 CPU 可训 ·  `GPU荐` 无显卡能跑但很慢 ·  `GPU需` 实际必须显卡
- **推理算力**：`CPU` / `GPU荐`（导出到行空板等边缘设备时只看推理这一栏）
- **平台状态**：`已支持` 现在就能在 app 内训练 ·  `易接入` 纯 sklearn、加几行即可 ·  `特征器` 只能当冻结特征提取器（不在课堂训练）·  `需后端` 要新增 PyTorch/ONNX 训练管线 ·  `引擎` 是现成推理引擎不是可训练分类器

> 「显卡检测灰置」逻辑建议：以**训练算力**为准。检测不到合适 GPU 时，把 `GPU需` 的卡片置灰禁选；`GPU荐` 的给黄色提示"可运行但较慢"；`CPU` 的正常可选。

每张卡片含四段说明：**作用 / 效能 / 简史 / 局限**。

## 一、图像分类 `image_classifier`

> 结构 = 特征提取（深度网络，重）+ 分类头（经典 ML，轻）。平台现在做的是「冻结 MobileNet 提特征 + 经典分类头」。

### A. 分类头（经典 ML，跑在特征向量上）

- **逻辑回归 Logistic Regression** · 训练`CPU` 推理`CPU` · `已支持`
  - 作用：给每个特征加权打分，算出属于各类别的概率。
  - 效能：快而稳，能输出概率；线性边界，多数迁移特征上已够用。
  - 简史：统计学经典（1958，Cox），机器学习最常用的基线分类器。
  - 局限：只能画"直"的边界，特征不好时易混。

- **K 近邻 KNN** · 训练`CPU` 推理`CPU` · `已支持`
  - 作用：不总结规则，预测时找最像的 K 个旧样本投票。
  - 效能：原理直观（物以类聚），在 MobileNet 这类有意义的特征上表现不错。
  - 简史：1951 提出、1967 正式化，最古老的模式识别方法之一。
  - 局限：样本多了预测变慢，怕无关特征。

- **随机森林 Random Forest** · 训练`CPU` 推理`CPU` · `已支持`
  - 作用：训练很多棵不同的决策树，预测时投票。
  - 效能：抗噪、不挑数据，通常比单棵树准。
  - 简史：2001 年 Breiman 提出，长期是表格/特征分类的强基线。
  - 局限：模型较大，没有单棵树直观。

- **支持向量机 SVM** · 训练`CPU` 推理`CPU` · `易接入`
  - 作用：找一条"间隔最大"的分界线把两类分开（核技巧可非线性）。
  - 效能：中小数据、清晰特征上精度高，泛化好。
  - 简史：1995 年 Cortes & Vapnik，深度学习兴起前的"霸主"算法。
  - 局限：大数据慢、需调核与正则，多分类要拆分。

- **梯度提升 GBDT（HistGradientBoosting）** · 训练`CPU` 推理`CPU` · `易接入`
  - 作用：一棵接一棵地训练小树，每棵纠正前面的残差。
  - 效能：表格/特征任务上常年精度最强之一；sklearn 自带、无需额外依赖。
  - 简史：1999/2001 Friedman 提出，XGBoost(2016)/LightGBM(2017) 发扬光大。
  - 局限：小数据易过拟合，参数较多。

- **多层感知机 MLP（浅层神经网络）** · 训练`CPU`(小)/`GPU荐`(大) 推理`CPU` · `易接入`
  - 作用：全连接神经网络，自动学非线性特征组合。
  - 效能：能拟合复杂关系，是"神经网络"概念的最小教学载体。
  - 简史：1986 反向传播（Rumelhart 等）让它实用化，神经网络复兴起点。
  - 局限：小数据易过拟合、不可解释、要调网络结构。

> 朴素贝叶斯不适用于图像：MobileNet 特征含负值，而它要求非负输入。

### B. 特征提取 / 端到端（深度卷积/Transformer，`特征器`）

- **LeNet-5** · 训练`CPU`(小) 推理`CPU` · `需后端`
  - 作用：最早的卷积网络，识别手写数字。
  - 效能：在 MNIST 级任务上够用，体量极小。
  - 简史：1998 年 Yann LeCun，奠定 CNN 基本范式。
  - 局限：太浅，复杂图像无力。

- **AlexNet** · 训练`GPU需` 推理`CPU`(慢) · `特征器`
  - 作用：深层 CNN 做大规模图像分类。
  - 效能：2012 ImageNet 夺冠、错误率断崖下降，引爆深度学习。
  - 简史：2012 Krizhevsky & Hinton，首个 GPU 训练的里程碑网络。
  - 局限：参数多、按今天看结构粗糙。

- **VGGNet** · 训练`GPU需` 推理`CPU`(慢) · `特征器`
  - 作用：用堆叠的 3×3 小卷积加深网络。
  - 效能：结构简单优雅，迁移特征至今仍被用作基线。
  - 简史：2014 牛津 VGG 组。
  - 局限：参数量巨大（>100M），慢。

- **GoogLeNet / Inception** · 训练`GPU需` 推理`CPU`(慢) · `特征器`
  - 作用：用 Inception 模块在同层并行多尺度卷积。
  - 效能：参数少、精度高，2014 ImageNet 冠军。
  - 简史：2014 Google。
  - 局限：结构复杂、不易改。

- **ResNet（残差网络）** · 训练`GPU需` 推理`CPU`(中) · `特征器`
  - 作用：引入"跳跃连接"让网络能堆到上百层。
  - 效能：2015 ImageNet 冠军，至今最常用的骨干网之一。
  - 简史：2015 何恺明等（微软亚研院）。
  - 局限：大版本较重，边缘端需选小版本(ResNet18)。

- **MobileNet v1/v2/v3** · 训练`GPU荐` 推理`CPU` · `特征器`（平台现用 v2）
  - 作用：用深度可分离卷积做轻量 CNN，专为手机/边缘。
  - 效能：体量小、CPU 推理快，精度/速度平衡好——边缘端首选。
  - 简史：Google，v1(2017)→v2(2018)→v3(2019)。
  - 局限：精度略低于大模型。

- **EfficientNet（含 -lite）** · 训练`GPU需` 推理`CPU` · `特征器`
  - 作用：用"复合缩放"同时放大深/宽/分辨率。
  - 效能：同精度下参数/算力更省，-lite 版适合边缘。
  - 简史：2019 Google（Tan & Le）。
  - 局限：训练对超参敏感。

- **Vision Transformer ViT** · 训练`GPU需` 推理`GPU荐` · `特征器`
  - 作用：把图像切块当序列，用 Transformer 做分类。
  - 效能：大数据下超过 CNN，是当前 SOTA 主流方向。
  - 简史：2020 Google，将 NLP 的 Transformer 引入视觉。
  - 局限：吃数据、吃算力，小数据不如 CNN。

- **ConvNeXt** · 训练`GPU需` 推理`GPU荐` · `特征器`
  - 作用：用 Transformer 的设计经验"现代化"纯 CNN。
  - 效能：精度对标 ViT 而保留 CNN 的高效。
  - 简史：2022 Meta。
  - 局限：仍偏重，边缘端不友好。

- **CLIP** · 训练`GPU需` 推理`CPU`(慢)/`GPU荐` · `特征器`
  - 作用：图文对比学习，可做零样本分类（不训练直接按文字标签分）。
  - 效能：泛化极强，无需训练即可识别新类别。
  - 简史：2021 OpenAI。
  - 局限：模型大，边缘端吃力。

## 二、文本分类 `text_classifier`

> 结构 = 文本向量化 + 分类头。平台现用 TF-IDF + 经典分类器。

### A. 向量化 + 经典分类头

- **TF-IDF + 逻辑回归 / 朴素贝叶斯 / 随机森林** · 训练`CPU` 推理`CPU` · `已支持`
  - 作用：统计词频构成向量，再用经典分类器判类。
  - 效能：中文短文本上又快又稳，是最实用的基线。
  - 简史：TF-IDF（1970s）+ 朴素贝叶斯（垃圾邮件过滤成名）。
  - 局限：不懂语序与语义。

- **朴素贝叶斯 Naive Bayes** · 训练`CPU` 推理`CPU` · `已支持`
  - 作用：统计每个词在各类出现频率，用概率反推类别。
  - 效能：极快，小数据强，关键词区分明显时尤佳。
  - 简史：1960s 起用于文本，垃圾邮件过滤经典。
  - 局限：假设词相互独立。

- **线性 SVM（LinearSVC）** · 训练`CPU` 推理`CPU` · `易接入`
  - 作用：在高维稀疏词向量上找最大间隔分界。
  - 效能：文本分类的经典强基线，常胜过逻辑回归。
  - 简史：SVM(1995) 在 2000s 文本分类中广泛使用。
  - 局限：不输出概率，多分类要拆。

- **fastText** · 训练`CPU` 推理`CPU` · `易接入`
  - 作用：词/子词向量 + 线性分类，兼顾词形。
  - 效能：训练飞快、效果好，工业界常用的轻量方案。
  - 简史：2016 Facebook。
  - 局限：语义理解仍有限。

### B. 深度学习（序列/预训练语言模型）

- **word2vec / GloVe（词向量）** · 训练`CPU`/`GPU荐` 推理`CPU` · `特征器`
  - 作用：把词映射为语义向量（"国王-男+女≈王后"）。
  - 效能：让下游模型理解词义相近，NLP 基石之一。
  - 简史：word2vec 2013（Google/Mikolov）、GloVe 2014（斯坦福）。
  - 局限：一词一向量，无法处理一词多义。

- **TextCNN** · 训练`CPU`(小)/`GPU荐` 推理`CPU` · `需后端`
  - 作用：用卷积在词向量上抓局部 n-gram 特征做分类。
  - 效能：短文本分类又快又好。
  - 简史：2014 Yoon Kim。
  - 局限：捕捉长距依赖弱。

- **RNN / LSTM / GRU** · 训练`GPU荐` 推理`CPU` · `需后端`
  - 作用：按顺序读文本，记住上下文。
  - 效能：擅长序列与长依赖，Transformer 出现前的主力。
  - 简史：LSTM 1997（Hochreiter & Schmidhuber）、GRU 2014。
  - 局限：训练慢、难并行，已被 Transformer 取代。

- **Transformer / BERT / DistilBERT / RoBERTa** · 训练`GPU需` 推理`CPU`(慢)/`GPU荐` · `特征器`
  - 作用：自注意力理解全局语义，预训练后微调到分类。
  - 效能：中文文本分类 SOTA；DistilBERT 是轻量蒸馏版。
  - 简史：Transformer 2017（Google）、BERT 2018（Google）。
  - 局限：训练要 GPU，CPU 推理偏慢，不适合课堂现训。

## 三、语音/音频分类 `audio_classifier`

> 结构 = 音频特征 + 分类头。平台现用手工梅尔频带特征 + 经典分类器。

### A. 特征 + 经典分类头

- **手工特征（MFCC / 梅尔频带）+ 逻辑回归/KNN/随机森林** · 训练`CPU` 推理`CPU` · `已支持`
  - 作用：把声波转成响度/音调分布等数值特征再分类。
  - 效能：环境音/关键词等任务上轻量够用。
  - 简史：MFCC 1980（Davis & Mermelstein），数十年语音特征标准。
  - 局限：手工特征上限有限。

- **SVM / GBDT / MLP（音频特征上）** · 训练`CPU` 推理`CPU` · `易接入`
  - 作用：在音频特征向量上做更强的分类头。
  - 效能：同图像头，GBDT/SVM 常优于线性模型。
  - 简史：见前。
  - 局限：仍受手工特征上限制约。

- **GMM-HMM** · 训练`CPU` 推理`CPU` · `需后端`
  - 作用：用高斯混合+隐马尔可夫建模语音时序。
  - 效能：深度学习前的语音识别标准方法。
  - 简史：1980s–2000s 语音识别主流。
  - 局限：已被神经网络全面超越。

### B. 深度音频模型（多作冻结特征提取器）

- **CNN（频谱图）** · 训练`GPU荐` 推理`CPU` · `需后端`
  - 作用：把音频转成频谱图当"图片"用 CNN 分类。
  - 效能：环境声/乐器分类效果好。
  - 简史：2015 起广泛用于音频。
  - 局限：要 GPU 训练才高效。

- **YAMNet / VGGish / PANNs** · 训练`GPU需` 推理`CPU` · `特征器`
  - 作用：在大规模音频上预训练的网络，当现成音频特征提取器。
  - 效能：YAMNet 基于 MobileNet、CPU 推理可行，比手工特征强很多。
  - 简史：VGGish 2017、YAMNet 2019、PANNs 2019/2020（均 Google/学界）。
  - 局限：训练需 GPU，平台只宜当特征器。

- **Wav2Vec2 / Whisper** · 训练`GPU需` 推理`GPU荐` · `特征器`/`引擎`
  - 作用：自监督语音表示 / 端到端语音识别。
  - 效能：语音识别 SOTA，Whisper 多语种强。
  - 简史：Wav2Vec2 2020（Meta）、Whisper 2022（OpenAI）。
  - 局限：重，CPU 推理慢，超出本平台课堂定位。

## 四、传感器决策 / 表格数据 `sensor_decision_model`

> 直接在数值表格上训练，无特征提取阶段。**表格任务上 GBDT 仍全面压制深度学习**（2024 多项基准）。

- **决策树 Decision Tree** · 训练`CPU` 推理`CPU` · `已支持`
  - 作用：自动学一串 if-else 规则分数据。
  - 效能：完全可解释——规则能直接读出，最适合教学。
  - 简史：CART 1984（Breiman）、ID3/C4.5 1986–1993（Quinlan）。
  - 局限：易过拟合，数据稍变规则大变。

- **随机森林 / KNN / 逻辑回归 / 朴素贝叶斯** · 训练`CPU` 推理`CPU` · `已支持`(RF/KNN) / `易接入`
  - 作用：树投票 / 近邻 / 线性概率 / 频率概率，多种思路对照。
  - 效能：随机森林是表格强基线，其余作教学对照。
  - 简史：见前。
  - 局限：各有所短（见各自卡片）。

- **支持向量机 SVM** · 训练`CPU` 推理`CPU` · `易接入`
  - 作用：最大间隔分界，可非线性核。
  - 效能：中小表格上稳健。
  - 简史：1995。
  - 局限：大数据慢、需缩放。

- **梯度提升 GBDT（XGBoost / LightGBM / CatBoost / HistGB）** · 训练`CPU`/`GPU荐` 推理`CPU` · `易接入`(HistGB) / `需依赖`(XGB等)
  - 作用：逐步纠错地集成小树。
  - 效能：**表格数据当前最强、竞赛最常胜**；HistGradientBoosting 是 sklearn 自带版，无需额外依赖。
  - 简史：XGBoost 2016（陈天奇）、LightGBM 2017（微软）、CatBoost 2017（Yandex）。
  - 局限：小数据易过拟合，参数较多。

- **多层感知机 MLP** · 训练`CPU`(小)/`GPU荐` 推理`CPU` · `易接入`
  - 作用：全连接神经网络拟合非线性。
  - 效能：表格上通常不如 GBDT，但可作"神经网络"教学。
  - 简史：1986 反向传播。
  - 局限：调参多、不可解释。

- **深度表格网络 TabNet / FT-Transformer / TabPFN** · 训练`GPU荐`/`GPU需` 推理`CPU`(慢) · `需后端`
  - 作用：为表格设计的神经网络（注意力/Transformer/先验拟合）。
  - 效能：少数大表格上能追平 GBDT；TabPFN 对小表格"免训练"推理新颖。
  - 简史：TabNet 2020（Google）、FT-Transformer 2021、TabPFN 2022。
  - 局限：多数情况仍打不过 GBDT，工程成本高。

## 五、智能问答 `qa_retrieval`（检索，不是分类器）

- **TF-IDF / BM25（关键词检索）** · 训练`CPU` 推理`CPU` · `已支持`(TF-IDF)
  - 作用：按词频匹配最相关的已知问句，返回其答案。
  - 效能：可靠、可控、毫秒级；BM25 是搜索引擎经典排序。
  - 简史：TF-IDF 1970s、Okapi BM25 1994。
  - 局限：只看字面，换种问法易漏。

- **词向量平均（word2vec/GloVe）检索** · 训练`CPU` 推理`CPU` · `易接入`
  - 作用：用词向量均值表示句子做相似度。
  - 效能：比纯关键词略懂语义。
  - 简史：2013–2014。
  - 局限：长句语义表达粗糙。

- **向量检索（Sentence-BERT / DPR + FAISS）** · 训练`GPU需`(微调) 推理`CPU`(可,略慢)/`GPU荐` · `特征器`
  - 作用：用预训练句向量做语义检索，FAISS 加速近邻搜索。
  - 效能：能匹配"意思相同、说法不同"的问句，明显强于关键词。
  - 简史：Sentence-BERT 2019、DPR 2020（Facebook）。
  - 局限：模型较大，需向量索引。

- **交叉编码重排 Cross-Encoder** · 训练`GPU需` 推理`GPU荐` · `特征器`
  - 作用：对候选答案逐一精排，精度更高。
  - 效能：检索质量最好，但慢。
  - 简史：2019 起随 BERT 普及。
  - 局限：重，通常只排前若干条。

- **LLM RAG（大模型检索增强）** · 训练`GPU需` 推理`GPU需`/API · `引擎`
  - 作用：检索到资料后让大模型组织自然语言回答。
  - 效能：体验最好、最灵活。
  - 简史：2020 RAG 论文（Facebook）后随大模型爆发。
  - 局限：需 GPU 或云 API，超出本平台离线定位。

## 六、文字查错 / OCR `ocr_typo_checker`（OCR 引擎 + 比对，非可训练分类器）

> 这里"算法"指**识别引擎**，查错是把识别结果与标准答案比对。

- **Tesseract** · 推理`CPU` · `引擎`
  - 作用：开源 OCR 引擎，识别图片中的文字。
  - 效能：英文好、轻量；中文一般。
  - 简史：1985 HP 研发，2005 Google 开源，v4(2018) 改用 LSTM。
  - 局限：复杂版式/中文场景精度有限。

- **EasyOCR**（平台导出包现用） · 训练`GPU需`(自训) 推理`CPU`(慢)/`GPU荐` · `引擎`
  - 作用：CRAFT 检测文字框 + CRNN 识别，开箱即用。
  - 效能：中英文都不错，社区流行。
  - 简史：基于 CRAFT(2019)+CRNN(2015)，2020 起流行。
  - 局限：CPU 上较慢。

- **PaddleOCR** · 推理`CPU`/`GPU荐` · `引擎`
  - 作用：DB 检测 + CRNN/SVTR 识别的中文 OCR 套件。
  - 效能：中文场景效果与速度俱佳，有超轻量版。
  - 简史：百度，2020 起，SVTR(2022) 识别器。
  - 局限：依赖较重。

- **TrOCR** · 训练`GPU需` 推理`GPU荐` · `引擎`
  - 作用：纯 Transformer 的端到端 OCR。
  - 效能：手写/印刷精度高，是 SOTA 方向。
  - 简史：2021 微软。
  - 局限：重，需 GPU。

- **识别子算法：CRNN / SVTR / Attention-OCR；检测子算法：CRAFT / DBNet** · `引擎`内部
  - 作用：分别负责"把一行字读出来"和"在图里框出文字"。
  - 简史：CRNN 2015、CRAFT/DBNet 2019、SVTR 2022。

> 备注：你举例的 **SuperPoint**（2018，Magic Leap）是自监督**关键点检测与描述**网络，用于图像匹配/SLAM，不属于上述六类分类/识别任务；这里按你想要的"详细到具体模型+效能+简史"的粒度来组织全集。若以后平台加入"图像关键点/特征匹配"能力，可单列一类收纳 SuperPoint、SIFT、ORB 等。

## 运行依赖审计

| 算法 / 能力 | 依赖 | 在本项目中的状态 |
| --- | --- | --- |
| 逻辑回归、朴素贝叶斯、随机森林、K近邻、决策树、梯度提升 GBDT、多层感知机 MLP、支持向量机 SVM | `scikit-learn` | **基础依赖**（`pip install -e .` 即装），8 个可训练算法**依赖已齐全** |
| MobileNet 图像特征提取（图像分类的特征器） | `onnxruntime` + `models_pretrained/mobilenetv2.onnx` | onnxruntime 是基础依赖；模型文件由 `python scripts/download_pretrained.py` 下载（已安装）。缺失则自动降级为像素特征 |
| SSD 目标检测（playground 演示） | `onnxruntime` + `models_pretrained/ssd_mobilenet.onnx` | 同上，缺失则降级为人脸检测 |
| OCR 文字识别（导出包内运行） | `easyocr`、`torch`、`opencv-python-headless` | `.[ocr]` 可选 extra（已在 pyproject 声明，体积大，按需安装） |
