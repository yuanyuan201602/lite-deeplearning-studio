## 0. 背景与目标

### 当前定位

本平台现阶段是一个「通用教育工作台」：帮 K12 学生在不写代码的前提下走完一个完整 AI 项目的四步流程（准备数据 → 训练模型 → 测试效果 → 导出材料）。竞赛功能已暂时移出首页，等通用教育模块成熟后再叠加。

### 教育部分的目标

> 用「做实验」代替「读课本」——学生操作真实数据、观察真实结果、形成对机器学习的直觉认知。

具体拆解：

1. **理解流程**：知道 AI 项目的每一步在做什么、为什么这样做。
2. **建立直觉**：亲眼看到"加更多数据后准确率上升"、"两类数据太像就分不开"等现象，而不是背这些结论。
3. **认识算法全貌**：不只会用平台提供的 CPU 算法，也知道深度学习是什么、有哪些主流方法、和传统 ML 有何不同。
4. **迁移能力**：走完通用练习后，能独立把同样的思路用到新问题——包括竞赛题目和真实场景。

### 不在此次 PRD 范围内

- 竞赛适配（比赛规则、提交清单）→ 后续单独 PRD
- 硬件部署教学（行空板 M10）→ 属于导出材料包的配套内容
- 大模型 / 生成式 AI 的原理教学 → 未来考虑

## 2. 现有教育内容盘点

### 2.1 已有的层次

| 位置 | 内容形式 | 覆盖的知识点 |
|---|---|---|
| 全局侧栏「新手教程」 | 6 章文字抽屉 | ML 是什么、四步流程、五种任务、术语表、报错排查、判断成功 |
| 任务介绍页 `concept_intro` | 1 段文字卡片 | 每个任务的核心原理（1–3 句话） |
| 项目页每步 `step_guides` | 每步 1 段文字折叠块 | 当前步骤的操作要点 + 对应理论 |
| 训练页「算法卡片」 | 卡片（文字）| 可训练算法（经典 CPU）+ 置灰深度模型，含中文讲解 |

### 2.2 缺口分析

下面按「学生要经历的认知层次」从浅到深列出目前**没有覆盖**的内容。

#### A. 直觉实验类（最缺，最有教学价值）

学生目前只能"做了一次"，没有工具引导他们**做对比实验**：

- 加数据前后的准确率对比
- 数据质量差 vs 好的结果差异
- 不同算法在同一份数据上的表现对比（已有 compare 功能，但缺教学引导）
- 选不同特征提取方式（MobileNet vs 像素）的结果差异
- 过拟合的直观演示（训练准确率高但交叉验证低是什么感觉）

#### B. 模型可解释性内容

- 决策树训练后会展示规则树，但没有人带学生"读懂这棵树"
- 图像分类：哪些区域影响了模型的判断？（热力图 / Grad-CAM，未来可选）
- 文本分类：哪些词权重最高？（TF-IDF 词权重可视化）

#### C. 深度学习 vs 经典 ML 的系统对比

算法卡片展示了深度模型，但没有讲：
- 神经网络的基本结构（层 → 激活 → 输出）
- 为什么图像/语音任务深度学习更强（特征层次）
- 迁移学习是什么：为什么用 MobileNet 比从头训练好
- 训练过程（损失函数 / 梯度下降）的直觉感受

#### D. 数据质量与工程

- 什么是好的标注
- 类别不平衡会怎样
- 训练集 / 验证集 / 测试集的分工（平台目前只有训练集 + 用户自己测）
- 数据增强的基本思路（对图像任务尤其重要）

#### E. 评估体系

- 准确率不是唯一指标（什么场景下精确率/召回率更重要？K12 水平举例即可）
- 混淆矩阵：模型容易把哪类搞混？
- 置信度分布：为什么大多数正确预测置信度高，错误预测置信度低？

## 4. 分模块内容规格

### 4.1 实验卡片（Experiment Cards）— 优先级 P1

**位置**：嵌入每个通用任务的项目页，作为独立的「实验」标签页，或在训练报告下方触发。  
**形式**：引导式对话——平台观察到学生的数据状态，主动抛出一个「你想不想试试……」的实验建议。

#### 实验清单（具体 + 可操作）

| 实验 | 触发条件 | 学生操作 | 平台呈现 | 教学结论 |
|---|---|---|---|---|
| E01 加数据实验 | 训练后准确率 < 80% | 补充若干样本，再次训练 | 前后准确率对比条形图 | 数据量直接决定模型上限 |
| E02 数据质量实验 | 有 ≥ 3 类样本 | 把某类一半样本挪到另一类当噪声，重新训练 | 准确率下降 + 提示原因 | 标注错误比数据少危害更大 |
| E03 算法赛跑 | 完成训练 | 点击「对比所有模型」 | 已有 compare 表格 + **新增**：逐行中文解读当前结果（谁赢了/为什么） | 不同算法适合不同数据形态 |
| E04 特征提取对比 | 图像任务 + 已下载 MobileNet | 切换「像素」和「MobileNet」分别训练 | 两次准确率并排 + 迁移学习一句话解读 | 预训练特征比原始像素信息量大 |
| E05 过拟合演示 | 决策树 / 小数据 | 刻意只用 2–3 张图片训练 | 训练准确率 100% 但交叉验证极低 + 解读文字 | 记住答案 ≠ 学会规律 |
| E06 类别边界实验 | 传感器 / 文本 | 给两类录入非常相似的数据 | 混淆矩阵（2×2）+ 高亮混淆的格子 | 特征区分度决定可分性 |
| E07 测试集采样实验 | 已导入整理数据集 | 点击「从测试集随机抽一条」多次 | 正确/错误的分布记录 + 累计正确率 | 单次测试不可靠，多次采样才客观 |

#### 内容格式规范（每张实验卡）

```
实验名称（10 字以内）
实验问题（一句话，以「如果……会怎样？」开头）
你需要做：[1–3 步操作]
看这里：[高亮的界面元素 / 数字]
我们发现：[结论，等学生操作后才显示]
为什么：[1 段原理解释，≤ 80 字]
```

### 4.2 深度学习专题页 — 优先级 P2

**位置**：新增一级页面 `/learn/deep-learning`，从首页「了解更多」或算法卡片「深入了解」跳入。  
**定位**：不是课本，是「地图」——让学生知道有哪些技术、各自解决什么问题，平台里的算法处于哪个位置。

#### 内容大纲

```
第 1 节 · 传统机器学习 vs 深度学习
  核心区别：特征提取是手工设计的 vs 网络自动学的
  适用场景：小数据/表格用传统 ML；大数据/图像/语音/文本用深度学习
  [示意图] 流水线对比：手工特征 vs 端到端

第 2 节 · 神经网络的基本结构（MLP）
  神经元 → 层 → 网络
  激活函数的直觉：让网络能画弯的分界线
  正向传播 / 反向传播（「试错调整」的直觉，不涉及公式）
  [动画/图示] 数据流过多层网络变成预测结果

第 3 节 · 图像任务为什么用 CNN
  局部感受野的直觉：「看一块区域」再拼整体
  卷积 = 滑动窗口找特征
  层次特征：浅层看边缘 → 深层看形状 → 最深层看语义
  本平台的位置：MobileNet 已帮你做了这一步（特征提取），你只需训练顶层分类器
  [图示] MobileNet 结构简化图；从像素到 1000 维特征向量的示意

第 4 节 · 文本任务的演进路线
  词袋 → TF-IDF（平台在用）→ 词向量（Word2Vec / fastText）→ LSTM → Transformer / BERT
  每个阶段解决了前一个阶段的什么问题
  [时间轴图]

第 5 节 · 音频任务
  声波 → 频谱图 → CNN 分类
  梅尔频谱：把声音的时频特征变成「声音图片」
  平台的方案：手工频带能量特征（轻量）；深度方案：频谱图 CNN / YAMNet

第 6 节 · 你在哪里 · 下一步去哪里
  本平台覆盖范围标注在技术地图上
  初学者路线：本平台 → Python + scikit-learn → PyTorch 入门 → 比赛 / 项目
  参考资源：中文推荐（不放外链，只描述书名/课程类别）
```

## 5. 媒体形式路线图

教学内容的信息密度和吸引力，随媒体形式递进。以下按实现难度排序，建议分阶段引入。

### 阶段 1（当前）· 文字 + 数字结果

- 优点：零额外开发，已全部实现
- 缺点：抽象概念靠学生自己脑补，互动性低

### 阶段 2（近期）· 静态图示 + 简单数据可视化

**适合的内容**：

| 内容 | 形式 | 实现方式 |
|---|---|---|
| 算法家族关系图 | SVG 树形图 | 内联 SVG，无依赖 |
| 训练准确率 vs 交叉验证准确率 对比条形图 | 两根并排的 bar | 纯 CSS/SVG，现有训练报告里已有数字 |
| 混淆矩阵（2×2） | 带颜色的格子 | CSS grid + 训练报告数据 |
| 词权重可视化（文本任务） | 高亮词语 | 前端字符串着色 |
| 特征提取流程图 | 横向示意图 | SVG / CSS Flex |
| 神经网络示意图（MLP） | 圆圈 + 连线 | SVG |
| MobileNet 流水线 | 横向步骤图 | SVG |
| 技术演进时间轴 | 横向时间轴 | SVG / CSS |

**实现原则**：全用浏览器原生技术（SVG + CSS），不引入图表库，保持零额外依赖。

### 阶段 3（中期）· 交互演示

**适合的内容**：

| 内容 | 交互形式 | 描述 |
|---|---|---|
| 数据量 vs 准确率曲线 | 滑块拖动 | 拖动「训练数据量」，看准确率曲线变化（用预算好的结果查表，不用真实训练） |
| 神经网络前向传播 | 点击激活 | 点击一个输入节点，看激活如何传播到输出 |
| 过拟合 / 欠拟合 | 切换示意 | 「数据 5 条 vs 100 条」，切换展示对应的分界线复杂度 |
| 决策边界可视化 | 二维散点 | 传感器 2 特征数据 → 展示不同算法画出的分界线 |

**实现策略**：用 `<canvas>` 或轻量 SVG 动画实现，预算好的「查表值」静态嵌入，不需要实时训练。

### 阶段 4（远期）· 视频 + 程序演示

| 内容 | 形式 |
|---|---|
| 「第一次训练」演示视频 | 3–5 分钟屏幕录制，展示从上传数据到拿到预测结果的完整过程 |
| 概念讲解短视频 | 每章 1–2 分钟，配合文字内容，由老师或动画呈现 |
| 可运行代码片段 | 嵌入 Jupyter-style 的只读代码块 + 输出，展示「同样的事情用 Python 怎么写」 |
| 行空板部署演示 | 视频：把导出包传到行空板后的实际运行效果 |

**注意**：视频托管成本高，且视频内容容易过时（平台 UI 更新后视频就失效）。建议等功能稳定后再录制。

## 7. 成功指标（定性）

> 平台教育内容算「做成了」的判断标准，由老师在课堂观察来验证：

1. **学生在训练完第一个模型后，能不被提醒地说出**：「准确率高但交叉验证低是因为数据少或太相似。」
2. **学生在准确率不理想时，第一反应是**：回去看数据，而不是反复点训练按钮。
3. **学生在选算法时，能说出一句理由**：不是随便选，也不是总选同一个。
4. **学生走完一个通用任务后，能自发问**：「深度学习的那些模型是怎么工作的？」
5. **老师能用本平台完整上一节 40 分钟的 AI 入门课**，不需要补充额外 PPT。
