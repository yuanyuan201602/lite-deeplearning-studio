# 智能博物轻量版学生培训文档

## 1. 课程目标

本培训文档指导学生使用"智能博物轻量版"在浏览器里完成全部 AI 训练，并导出比赛材料包。

完成本课程后，学生应能做到：

- 在 Windows 学生机上安装并启动智能博物轻量版。
- 用四步工作流（准备数据 → 训练模型 → 测试效果 → 导出材料）完成每个挑战。
- 用样例数据或自己采集的数据在浏览器内完成 AI 训练与测试。
- 导出含训练好模型的比赛材料包，并迁移到行空板 M10。
- 将项目迁移到行空板 M10，并连接 DFRobot 开源硬件外设完成展示。

统一硬件基线：

- 主控：行空板 M10
- 语音播报：DFRobot 语音合成模块或兼容开源语音外设
- 图像/卡片输入：行空板 M10 摄像头或兼容 DFRobot 摄像头模块
- 展示输出：行空板 M10 屏幕

## 2. 课前准备

### 2.1 软件准备

学生电脑需要：

- Windows 10 或 Windows 11
- Python 3.11 或 3.12
- 浏览器：Edge 或 Chrome
- 智能博物轻量版安装包：`SmartMuseum-Windows-Setup.zip`

建议老师提前把安装包发到学生电脑桌面。

### 2.2 Python 检查

按 `Win + R`，输入 `cmd`，打开命令行，输入：

```bat
python --version
```

如果能看到 `Python 3.11.x` 或 `Python 3.12.x`，说明 Python 可用。

如果提示找不到 Python，需要重新安装 Python，并勾选：

```text
Add python.exe to PATH
```

### 2.3 硬件准备

每组建议准备：

- 行空板 M10 一块
- USB 数据线
- DFRobot 语音合成模块
- 摄像头模块或行空板 M10 可用摄像头
- 若干非遗人物卡片、知识卡片、词语卡片、句子卡片
- 电脑与行空板在同一课堂环境中可拷贝文件

## 3. 安装智能博物轻量版

### 3.1 解压安装包

1. 找到 `SmartMuseum-Windows-Setup.zip`。
2. 右键选择"全部解压"。
3. 进入解压后的 `lite-deeplearning-studio` 文件夹。

不要直接在压缩包预览窗口里双击脚本，必须先解压。

### 3.2 一键安装

双击：

```text
一键安装.bat
```

安装脚本会自动：

- 创建本地运行环境 `.venv`
- 安装网页应用依赖
- 安装轻量 AI 依赖（scikit-learn、numpy、joblib、pillow）
- 创建桌面快捷方式

安装完成后，窗口会提示可以启动。

macOS 用户双击：

```text
一键安装.command
```

### 3.3 OCR 增强安装

如果要完成"挑战二：了解非遗专业知识"的 OCR 错字识别，建议双击：

```text
安装OCR增强.bat
```

OCR 增强会安装 EasyOCR、Torch 和 OpenCV，下载时间更长。

如果课堂网络较慢，可以先完成其他挑战，再由老师统一安排 OCR 安装。

### 3.4 启动软件

双击：

```text
启动软件.bat
```

macOS 用户双击：

```text
启动软件.command
```

或双击桌面快捷方式：

```text
智能博物轻量版
```

浏览器打开：

```text
http://127.0.0.1:8000
```

正常情况下，首页只会显示"智能博物"相关任务，不会显示"优创未来"。

## 4. 软件使用总流程

每个挑战都在浏览器中按四步完成：

```
准备数据  →  训练模型  →  测试效果  →  导出材料
```

**完整操作步骤：**

1. 打开首页，选择挑战任务卡片。
2. 填写项目名称和学生姓名，确认目标硬件为"行空板 M10 + DFRobot 开源硬件外设"。
3. 点击"创建项目，开始训练"，进入四步工作流页面。
4. **第 1 步「准备数据」**：在浏览器中录入数据（上传图片/粘贴文本/输入问答对/粘贴传感器表格/输入正确文字），点击"保存数据"。
5. **第 2 步「训练模型」**：点击"开始训练"，模型在电脑本地训练，数秒内完成，页面显示准确率和分类统计。
6. **第 3 步「测试效果」**：输入一条新数据（或上传一张新图片）点击测试，实时看到模型预测结果。
7. **第 4 步「导出材料」**：点击"导出比赛材料包"，下载 zip 文件，包含训练好的模型、全部数据、可运行脚本、硬件说明和比赛清单。
8. 解压导出包，在电脑端运行 `python run.py` 验证。
9. 查看 `outputs/ai_validation_result.json`，确认 `"status": "ok"`。
10. 按 `hardware/README.md` 迁移到行空板 M10，连接 DFRobot 外设展示。
11. 整理 `submission/README.md` 中列出的提交材料。

> **关键点**：AI 模型在浏览器里训练好后会随导出包一起打包，不需要在电脑命令行单独运行训练脚本。

## 5. 挑战一：认识非遗传承匠人

### 5.1 比赛目标

识别人脸卡片或人物卡片，显示并播报人物信息。

常见输出格式：

- 小学：`这是XXX`
- 初中/高中：`这是XXX，XXX`

### 5.2 选择任务

打开首页，点击：

```text
挑战一：认识非遗传承匠人
```

### 5.3 第 1 步：准备数据

进入项目页后，在「准备数据」面板：

- 点击"添加类别"，输入人物名称（如"京剧传承人"、"剪纸传承人"、"陶艺传承人"）。
- 为每个类别上传 3 到 5 张清晰人物卡片图片（JPG/PNG，建议 200KB 以内）。
- 所有类别上传完毕后，点击"保存数据"。

> 每个类别至少需要 2 张图片才能开始训练。

### 5.4 第 2 步：训练模型

切换到「训练模型」面板，点击"开始训练"：

- 训练通常在 5 秒内完成。
- 页面会显示训练准确率和各类别样本数量。
- 准确率低于 80% 时建议每类再增加 2 到 3 张图片，重新保存后再次训练。

### 5.5 第 3 步：测试效果

切换到「测试效果」面板：

- 上传一张新的人物卡片图片（不在训练集内）。
- 点击"测试"，查看模型预测的人物类别和置信度。
- 确认预测符合预期后进入下一步。

### 5.6 第 4 步：导出材料

切换到「导出材料」面板：

- 点击"导出比赛材料包"，等待打包完成后下载 zip。
- 解压后检查：
  - `models/image_classifier.joblib`（已训练的模型）
  - `data_sample/images/`（各类别图片数据）
  - `predict.py` / `run.py`
  - `speech/speech_output.py`（DFRobot 语音播报模板）
  - `docs/competition_checklist.md`（挑战要求核对清单）

### 5.7 电脑端验证

在任务包目录中打开命令行，运行：

```bat
python run.py
```

成功后检查 `outputs/ai_validation_result.json` 中 `"status": "ok"`。

### 5.8 迁移到行空板 M10

迁移重点：

- 使用行空板 M10 摄像头拍摄人物卡片。
- 将拍摄图片保存到预测图片目录，调用模型预测人物类别。
- 将识别结果显示在行空板屏幕上。
- 调用 DFRobot 语音合成模块播报结果。

演示时确认：卡片在摄像头画面中清晰，播报格式符合比赛要求。

## 6. 挑战二：了解非遗专业知识

### 6.1 比赛目标

识别知识卡片文字，找出错别字位置，并播报更正结果。

### 6.2 选择任务

打开首页，点击：

```text
挑战二：了解非遗专业知识
```

### 6.3 第 1 步：准备数据

在「准备数据」面板输入正确的知识卡片文本：

```text
保护为主抢救第一合理利用传承发展
```

- 这是正确的参考文本，OCR 识别结果将与此比对。
- 如果已安装 OCR 增强，还可以上传知识卡片照片直接进行比对测试。

### 6.4 第 2 步：训练模型

点击"开始训练"，系统会保存正确文本并准备比对逻辑，无需大量数据，几秒内完成。

### 6.5 第 3 步：测试效果

在测试框输入一段包含错别字的文本，例如：

```text
保护为王抢救第一合理利用传承发展
```

系统会标出错别字位置并给出更正播报。

### 6.6 第 4 步：导出材料

- 点击"导出比赛材料包"，下载 zip。
- 解压后运行 `python run.py`，检查 `outputs/ai_validation_result.json`。
- 如果看到 `"status": "ocr_environment_missing"`，需要运行 `安装OCR增强.bat` 安装 EasyOCR。

### 6.7 迁移到行空板 M10

迁移重点：

- 用摄像头拍摄知识卡片，图片正对、无遮挡、光线均匀。
- OCR 识别后与正确文本比对，屏幕显示错字位置。
- DFRobot 语音合成模块播报更正结果。

## 7. 挑战三：非遗文化分类学览

### 7.1 比赛目标

训练并调用自建文本分类模型，判断词语属于哪一类非遗。

输出格式：

```text
XXX属于XXX
```

### 7.2 选择任务

打开首页，点击：

```text
非遗词语分类工作流
```

### 7.3 第 1 步：准备数据

在「准备数据」面板，为每个非遗类别添加词语样例：

- 点击"添加类别"，输入类别名（如"传统戏剧"、"传统美术"、"传统技艺"）。
- 每个类别至少输入 3 条样例词语或句子，每行一条。

示例数据：

| 类别 | 样例词语 |
|------|----------|
| 传统戏剧 | 昆曲、京剧、川剧 |
| 传统美术 | 剪纸、年画、泥塑 |
| 传统技艺 | 陶瓷烧制、刺绣、木雕 |

### 7.4 第 2 步：训练模型

点击"开始训练"：

- 模型使用 TF-IDF 和分类算法在本地训练，通常 2 秒内完成。
- 页面显示训练准确率（建议 ≥ 85%）。
- 准确率偏低时，增加各类别样例数量后重新训练。

### 7.5 第 3 步：测试效果

在测试框输入一个未出现在训练集中的词语，例如"皮影戏"，确认模型能预测出正确的非遗类别。

### 7.6 第 4 步：导出材料

- 点击"导出比赛材料包"，下载 zip。
- 解压后检查 `models/text_classifier.joblib`（已训练的模型）。
- 运行 `python run.py`，检查预测结果是否符合 `XXX属于XXX` 格式。

### 7.7 迁移到行空板 M10

- 将现场词语卡片转成文本输入，调用模型分类。
- 如需摄像头识别文字，可结合 OCR 流程。
- DFRobot TTS 播报分类结果。

## 8. 挑战四：非遗文化深化认知

### 8.1 比赛目标

识别句子卡片，判断句子中的非遗类别和名称。

播报格式：

```text
这是非遗XXX类别的XXX
```

### 8.2 选择任务

```text
挑战四：非遗文化深化认知
```

### 8.3 第 1 步：准备数据

在「准备数据」面板，为每个类别输入句子样例：

|------|----------|

### 8.4 第 2 步：训练模型

### 8.5 第 3 步：测试效果

### 9.1 比赛目标

围绕非遗传承与宣传做一个不重复常规挑战的互动作品。

平台提供的是轻量问答检索模板（基于 TF-IDF 相似度匹配）。

### 9.2 选择任务

打开首页，点击：

```text
非遗知识问答模板
```

### 9.3 第 1 步：准备数据

在「准备数据」面板，每行输入一组问答对，问题和答案用 Tab 分隔：

```text
什么是昆曲	昆曲是中国传统戏剧之一，被称为百戏之祖。
为什么要保护非遗	保护非遗可以传承文化记忆，让传统技艺继续被看见。
```

### 9.4 第 2 步：训练模型

点击"开始训练"，系统会建立问答索引，通常 1 秒内完成。

### 9.5 第 3 步：测试效果

在测试框输入一个问题，确认系统能返回最相关的答案。

### 9.6 第 4 步：导出材料

- 点击"导出比赛材料包"，下载 zip。
- 运行 `python run.py` 验证，检查问答检索结果。

### 9.7 展示建议

行空板 M10 屏幕显示问题和答案，DFRobot TTS 播报答案。

创意拓展可以加入：

- 非遗知识问答

## 12. 常见问题

### 12.1 打不开网页

检查 `启动软件.bat` 窗口是否还在运行。

如果端口被占用，可以让老师改用：

```bat
python scripts/start_studio.py --edition smart_museum --port 8010
```

### 12.2 OCR 任务失败

先确认是否运行过：

```text
安装OCR增强.bat
```

如果仍失败，先完成其他挑战，把 OCR 作为老师统一调试项。

### 12.3 识别不准

常见原因：

- 每类图片太少（建议每类 ≥ 5 张）。
- 图片太暗或模糊。
- 类别名称太接近。
- 训练数据和现场卡片差异太大。

解决方法：

- 每类增加 3 到 5 张清晰图片，重新保存数据后点击"开始训练"。
- 保持拍摄角度一致。
- 文本分类任务增加更多样例词语或句子。

### 12.4 语音不播报

检查：

- DFRobot 模块是否通电。
- 串口是否连接正确（设备管理器中确认 COM 号）。
- 程序中的串口号是否和实际设备一致。
- 电脑端无硬件时会用文本打印代替真实播报。

### 12.5 导出包里没有模型文件

必须先完成"第 2 步：训练模型"，点击"开始训练"并看到准确率后，再进行第 4 步导出。未训练时导出包不含模型。

## 13. 最终目标

完成训练后，学生应该能展示：

- 行空板 M10 拍摄或读取卡片。
- 本地 AI 模型完成识别或分类。
- 屏幕显示比赛要求格式。
- DFRobot 外设完成语音播报。
- 提交材料完整，能说明作品设计和 AI 技术点。
